# MLP Spec — "Claude Code의 파워를 터미널 없이" (2026-07-20)

> 근거 결정: **D54–D58** (decision-log.md). PRD(human-only) 반영 전까지 이 문서가 MLP 스코프의
> source of truth. MVP 개념은 D55의 합격 기준으로 대체됨.

---

## 1. Wedge (D54)

- **카테고리:** supervision 레이어 — AI가 코딩하면서 인간의 일은 작성→감독으로 이동, 감독에는 관찰
  인터페이스가 필요. Orca(22.7k★)가 카테고리 수요를 증명했으나 그들의 답은 여전히 터미널(파워 개발자용).
- **청중:** 에이전트를 감독해야 하지만 터미널/API키에는 평생 안 들어갈 사람 — **Lovable 천장 졸업자**
  (딸깍으론 부족해졌지만 터미널은 무서운 비개발자 빌더) ∪ CC-aware builder(D47).
- **판타지:** 경영시뮬 — **"AI 직원을 둔 사장님"**. RTS 유닛 컨트롤이 아니라 비게이머에게도 읽히는
  대중 판타지. (실측: Joshua가 자발적으로 이 롤플레이를 수행 — §6)
- **비즈니스 모델:** 호스티드 + 크레딧(D46) 유지 — 이 청중에겐 "설치 없음, 키 없음, 카드만"이 접근성.
- **껍데기:** 웹 퍼스트 유지, Electron 안 감(D57). 리라이트 기각 — 기존 아키텍처에 얹는다(§4 재사용 맵).

## 2. 페르소나

비개발자(또는 코드 안 읽는) 솔로 빌더. 만들고 싶은 실제 제품이 있고(스토어, SaaS, 커뮤니티 툴),
Lovable류로 시작해봤거나 CC를 만져봤고, 결과물이 "진짜 돈 버는 제품"이 되길 원한다. 코드가 아니라
결과물을 가리키며 피드백하고("여기 이거"), 에이전트가 일하는 동안 자리에 없다.
**파운더 본인이 이 페르소나의 실존 표본** — 도그푸딩이 곧 검증(D55).

## 3. 합격 기준 (D55, 북극성 테스트)

> **"비개발자인 파운더가 Claude Code를 한 번도 안 열고, pondas만으로 실제 도메인이 걸린 SaaS를
> 출시하고 2주간 운영(버그픽스 + 기능추가)할 수 있다."**

- 통과에 필요한 것 = 전부 MLP 스코프. 필요 없는 것 = 전부 컷.
- 타임라인 캘리브레이션: 데모 품질 며칠 / 합격 통과 몇 주(검증·외부 연동 지연·파운더 리뷰 병목 +
  기준 자체의 2주 운영 하한).
- 통과 시점의 스토리("개발자 아닌 내가 pondas만으로 출시했다") = 런치 콘텐츠.

- **게임 요소 필터(D54):** 모든 게임 요소는 감독 정보 또는 개입 수단을 나른다. 미니맵=주의 라우팅,
  핑=needs-input, 자원 카운터=실시간 크레딧. 정보 없는 장식만 컷.
- **온보딩(D58):** 첫 실행 경험 — 예시 목표 제시/첫 태스크 유도. "뭘 시킬지"의 빈 캔버스 해소.
  \+ 최소 제품 애널리틱스(방문/퍼널 계측).

## 5. 컷 리스트 (D56)

- AI 관광객용 요소 — "에이전트 처음 구경하는 사람" 대상 문은 D47/D54로 닫힘.
- 정보 없는 장식 — D54 필터 미통과 게임 요소.
- 제품빌딩 외 팀 유형 확장 — 한 페르소나, 한 job, 한 루프.
- 자체 하네스 자부심 — 매니지드 런타임(CMA/Agent SDK)으로 신뢰성 산을 낮출 수 있으면 그쪽.
  경쟁력은 하네스가 아니라 그 위 레이어.

## 6. 실측 근거 (2026-07-20 프로드 DB)

- **퍼널:** 오가닉 가입 6 → 프로젝트 생성 6 → 태스크 실행 1 → 복귀 0 → 결제 0.
  가입만 한 5명도 프로젝트명은 실의도(NeuroNex, Linesage, JM Group). 방문은 계측 안 됨(애널리틱스 부재).
- **Joshua 여정 (유일한 실행 유저, 30분 세션):** 스페인어권 비개발자, "피아트+크립토 결제 온라인 스토어"
  목표. 텍스트 에이전트(PM/리서처) 전부 성공, **빌드 에이전트(디자이너) 3연속 실패**(40-step ×2 +
  tenacity env 버그) — 눈에 보이는 결과물 0. 진행 알림 요청("avisa me...") → 오케가 지킬 수 없는 약속
  ("Te aviso") — B3 정직성 문제에 실유저가 걸림(#95로 수리됨, Joshua 이후). 사장님 롤플레이
  ("para eso le estoy pagando") 후 이탈. 300크레딧 소진.
- **함의:** UX 판타지 검증됨 / 이탈 원인 = 버킷③ / Joshua의 3가지 요구(알림·완성·결제되는 스토어) =
  버킷⑤③② 1:1 매핑 / 수리 후 재접촉 1순위(이메일 보유).

## 7. 빌드 오더

1. **버킷①② (깃헙 + 배포)** — 장난감→제품 경계. 없으면 wedge 불성립.
2. **버킷③ churn 머신 수리** — 40-step 예산제 + env 안정화 (Joshua 직접 사인).
3. **버킷④⑤ + 온보딩/애널리틱스** — 기존 기반 위 개선 연속.
4. 파운더가 합격 기준 테스트를 직접 수행(실 SaaS 1개 출시 + 2주 운영) → 통과 = 런치.

## 8. 리스크 (눈 뜨고 안고 가는 것)

- **양면 협공:** Lovable은 위로(컨트롤 확장), CC/Cursor는 아래로(친화적 앱), Orca도 쉬운 모드 가능 —
  wedge 유효기간이 짧을 수 있다. 대응 = 속도 + 청중 밀착(둘 다 incumbent DNA가 아닌 것).
- **신뢰성 산:** 비개발자를 섬길수록 기준 상승(그들은 못 고침). 대응 = 매니지드 런타임 적극 활용 +
  모델 개선 추세에 베팅.
- **바이럴 ≠ 리텐션:** 오피스 데모는 유입 자산일 뿐 — closure(D48)와 버킷①②가 리텐션을 만든다.
