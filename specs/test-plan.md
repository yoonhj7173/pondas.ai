# Test Plan — Phase 3 MLP (2026-07-20)

> 근거: D55(합격 기준), D56(버킷), implementation-plan 35–41. 원칙: **실행 검증 우선** — 빌드 성공/유닛 그린은 필요조건일 뿐, 항목 완료 판정은 라이브 인프라(로컬 docker 스택 or 프로드) 실동작으로만. LIVE 제품이므로 빌링 경로는 전 단계에서 무접촉 검증.

## 1. 유닛/통합 (pytest — PR마다 CI 게이트)

- 기존 스위트 유지(회귀 베이스라인). 신규 커버리지:
  - **GitHub 파이프:** push_version 재시도/실패 시 태스크 무영향(비동기 격리), 사람말 라벨 폴백, Restore = 새 버전 컷(히스토리 불변), 백필 순서.
  - **Deploy:** DeployProvider 어댑터 목킹 계약 테스트(provision/deploy/domain/db/env), 시크릿 암호화 라운드트립 + **로그 마스킹**(시크릿 값이 로그 포매터를 통과 못 함을 단정).
  - **예산제:** 토큰/시간 예산 초과 → 부분 결과 + continuation 저장 + 사람말 사유(조용한 실패 금지 단정); 컴팩션 트리거(70%)와 압축 후 컨텍스트 보존.
  - **푸시:** 구독 저장/삭제, 발송 채널이 기존 notification 파이프에 추가되고 실패해도 인앱 알림 무영향.
- 로컬 스택: `cpm-test-pg` + `cpm-test-redis` (docker), `APP_ENV=test`, alembic + seed.

## 2. E2E (Playwright — 페이즈 완료마다)

- **코어 플로우(회귀):** 가입 → 온보딩 v2 → 첫 태스크 → 결과 인패널 → 프리뷰/시어터 → 반복("make X blue") → 버전 증가.
- **Phase 3 신규:** History 패널 → Restore → 프리뷰 갱신; 배포(어댑터 목킹) → deploy_status SSE → Grand Opening 카드; 디자인 코멘트(요소 클릭 → 태스크 디스패치 payload 단정); 코드뷰 열람.
- 외부 API(GitHub/Vercel/Neon)는 E2E에서 목킹, **어댑터 실물 검증은 §4 스모크에서 별도**.

## 5. 적대적 테스트 (신규 표면 한정 — 항목 37/39 완료 시)

- **Deploy/시크릿:** 타 유저 프로젝트 배포/도메인/시크릿 접근 404; 시크릿 값이 SSE·로그·LLM 프롬프트·에러 메시지에 평문 노출 0(grep 스윕); 도메인 검증 우회(타인 도메인 연결) 불가; 악성 빌드 스크립트는 Vercel 샌드박스 안에서만.
- **GitHub:** installation token 스코프 최소(다른 리포 접근 불가); 콜백 위조/CSRF; 리포명 인젝션.
- **디자인 코멘트:** 프리뷰 주입 스크립트가 부모 DOM 접근 불가(postMessage origin 검증); selector/코멘트 문자열의 프롬프트 인젝션 → 시스템 룰 우선 확인.
- **푸시:** 구독 endpoint 위조로 타 유저 알림 수신 불가.
- 기존 XSS/SQLi/입력 코퍼스(ADVERSARIAL_TEST_RESULTS.md) 신규 입력면(코멘트, 도메인, 시크릿 키명)에 재실행.

## 6. 합격 테스트 프로토콜 (D55 — 최종 게이트, 항목 41)

1. **대상:** 파운더가 실제로 원하는 SaaS 1개(장난감 금지 — 실사용 의도가 있는 것).
2. **규칙:** Claude Code/터미널/에디터 사용 금지. 모든 작업은 pondas UI로만. 위반 시 해당 일자 무효.
3. **범위:** 아이디어 → 빌드 → 커스텀 도메인 출시 → **2주 운영**: 최소 버그픽스 5건 + 기능추가 3건 + 모바일(푸시 경유) 응답 3회.
4. **기록:** `specs/acceptance-run.md`에 일자별 요청→결과→막힌 곳→우회 여부. 막힘 = 즉시 이슈화 → 항목 35–40으로 환류.
5. **통과 기준:** 3의 전 항목 완료 + "CC였으면 더 빨랐을 것" 항목이 치명 아님(주관 판정 기록) + 운영 2주차에 신규 P0 버그 0.
6. **통과 후:** 운영 로그 = 런치 콘텐츠 원고; GTM 개시.
